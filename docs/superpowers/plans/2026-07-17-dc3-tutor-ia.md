# DC-3 y SEP en el Tutor IA — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

> **ACTUALIZADO 2026-07-17** por las dos decisiones de David sobre el spec:
> 1. **El SEP se rige por la misma regla que la DC-3.** Ambos certificados de pago (`dc3` **y** `sep`) se admiten por *curso marcado + intento de quiz aprobado*. El `diploma` sigue gratis/automático como hoy (y de hecho no se emite por este endpoint: se crea al momento de emisión del admin).
> 2. **Las solicitudes de certificado se anclan a Studio, no al catálogo legacy.** `certificate_requests.course_id` (FK a `courses.id`) se **reemplaza** por `studio_course_slug` (FK a `studio_courses.slug`). Con cero filas existentes (David, 2026-07-17), es un cambio destructivo seguro; repara de raíz que los ~20 cursos solo-Studio no pudieran anclar solicitud.

**Goal:** Que la **DC-3 y el SEP** se puedan solicitar desde el Tutor IA si y solo si el curso está marcado elegible para ese certificado (`studio_courses.dc3_available` / `studio_courses.sep_available`) y existe un intento de quiz **aprobado y calificado por el servidor**; que el Aula Virtual no pueda emitir **ninguno** de los dos por construcción; y que la solicitud se ancle al curso de Studio, no al catálogo legacy.

**Architecture:** Toda la lógica de decisión vive en módulos puros y testeables (`shared/quiz-grading.ts`, `shared/cert-eligibility.ts`, `shared/quiz-cooldown.ts`) y en un orquestador con dependencias inyectadas (`server/lib/studio-quiz-submit.ts`); las rutas Express solo traducen a HTTP. El submit de quiz califica **siempre** contra el quiz almacenado en el servidor y escribe un renglón append-only en la nueva tabla `studio_quiz_attempts`. `POST /api/me/certificates` cambia su criterio de admisión para **ambos** certificados de pago (studio marcado + intento aprobado) y **cambia su ancla** a `studio_course_slug`, sin tocar la plomería de cobro de F0 (checkout, webhook, duplicados, precio). La pestaña Certificado consume un endpoint nuevo (`GET /api/studio/courses/:slug/certificates`) que devuelve el estado real de DC-3 **y** SEP.

**Tech Stack:** Express 5 + Drizzle ORM (Postgres/Supabase) + Vite/React 18 + TanStack Query + vitest (node env, sin DB) + TypeScript strict. Español-MX en todo el copy y en los mensajes de error.

## Global Constraints

- TDD estricto por paso: test que falla → correrlo y verlo fallar → implementación mínima → correrlo y verlo pasar → commit. Nada de código sin test que lo cubra.
- Suite actual: **59/59 pasando**. Cada tarea debe terminar con `npx vitest run` verde y `npx tsc --noEmit -p tsconfig.json` limpio.
- La lógica pura va en su propio módulo, al estilo de `shared/cert-pricing.ts` / `shared/cert-duplicate.ts` (funciones puras, sin I/O, con su `*.test.ts` al lado). Los tests corren en `environment: "node"` **sin base de datos**: todo lo testeable se testea con funciones puras o con dependencias inyectadas (fakes).
- La calificación **siempre** se calcula en el servidor contra el quiz almacenado. El `score` del cliente se ignora por completo (no se acepta ni como sugerencia). Si no hay quiz almacenado que calificar, el intento **falla con error explícito** — nunca se auto-aprueba.
- Calificación mínima: se lee de `studio_quizzes.passing_score` (default de schema **70**). Prohibido el literal `70` en la ruta; el único `70` permitido es la constante `DEFAULT_PASSING_SCORE` documentada como espejo del default del schema.
- Espera entre intentos reprobados: **24 h**, validada **en el servidor**. Un intento **aprobado nunca es bloqueado** por cooldown. Intentos ilimitados, con espera.
- Regla de elegibilidad (idéntica para `dc3` y `sep`): `studio_courses.<cert>_available = true` **y** un intento aprobado, calificado por el servidor, de ese socio para ese curso. **No mira inscripciones, no mira módulos, no mira progreso.** El flag por certificado es `dc3_available` para DC-3 y `sep_available` para SEP.
- Tabla de intentos **append-only**: nunca se sobrescribe ni se actualiza un intento.
- Sin degradación silenciosa: si la consulta de elegibilidad falla, se registra con `console.error` y se devuelve un estado de error explícito (503). Jamás se asume "no elegible" en silencio.
- El precio **nunca** viene del cliente: siempre `CERT_PRICES_MXN` (`shared/cert-pricing.ts`). DC-3 = **$499 MXN**, SEP = **$1,999 MXN**.
- **No tocar la plomería de pago de F0**: `POST /api/certificates/webhook`, `decideCertTransition`, `isBlockingDuplicateStatus`, los estados `pending_payment` → `solicitado`, ni `resolveCertPriceMxn`. Solo cambian **a quién se deja pasar** y **a qué se ancla la solicitud**.
- **`POST /api/me/certificates` solo admite `dc3` y `sep`** (validado hoy en `certificates.ts:58`). El `diploma` no pasa por este endpoint: se crea al emitir (admin, `certificates.ts:231`) y **sigue gratis/automático**, sin cambios. Por eso la validación legacy `course_users.completed >= 100` y `getQuizByCourse` **se elimina por completo** de este endpoint: ya no queda ningún camino de pago colgado del catálogo legacy.
- **Ancla de la solicitud (nuevo):** `certificate_requests.studio_course_slug` es FK a `studio_courses.slug` (Task 8). El viejo `course_id` (FK a `courses.id`) **se elimina**. Consecuencia deliberada: cualquier curso del Tutor IA puede anclar una solicitud, incluidos los ~20 solo-Studio. Ya **no existe** el estado `sin_curso_legacy` del diseño anterior: era el parche de un problema que este cambio elimina de raíz.
- Sin migración de **datos**: nadie ha solicitado ni pagado un certificado (David, 2026-07-17). Verificable con `select status, count(*) from certificate_requests group by status;` (debe dar 0 filas). Esto es lo que hace seguro el cambio destructivo de FK en Task 8.

```ts
// Calificar un intento de quiz es una función pura y vive AQUÍ, no en la ruta.
// Regla del spec 2026-07-17 (P1): la calificación se calcula SIEMPRE en el
// servidor contra el quiz almacenado; el `score` que manda el cliente se ignora
// por completo. Si no hay quiz que calificar, esto LANZA: nunca se auto-aprueba.

/** Default del schema: `studio_quizzes.passing_score` y `course_quizzes.passing_score` son `NOT NULL DEFAULT 70`. */
export const DEFAULT_PASSING_SCORE = 70;

export interface GradableQuestion {
  correctIndex: number;
}

export interface QuizGrade {
  score: number;
  total: number;
  scorePercent: number;
  passed: boolean;
}

export function gradeQuizAttempt(
  questions: GradableQuestion[],
  answers: number[],
  passingScore: number = DEFAULT_PASSING_SCORE,
): QuizGrade {
  if (!questions || questions.length === 0) {
    throw new Error("No hay quiz almacenado que calificar: el intento no se puede aprobar.");
  }
  const total = questions.length;
  let score = 0;
  for (let i = 0; i < total; i++) {
    if (answers[i] === questions[i].correctIndex) score++;
  }
  const scorePercent = Math.round((score / total) * 100);
  return { score, total, scorePercent, passed: scorePercent >= passingScore };
}
```

```ts
import { gradeQuizAttempt, type GradableQuestion, type QuizGrade } from "@shared/quiz-grading";

// Orquestador del submit de quiz del Tutor IA. Vive fuera de la ruta y recibe
// sus dependencias inyectadas para que la regla —la única que separa un
// certificado federal de un `curl`— se pueda testear sin base de datos.
//
// Spec 2026-07-17, P1: el `score` del cliente NO se lee. Ni como sugerencia.

export interface StoredQuiz {
  questions: GradableQuestion[];
  passingScore: number;
}

export interface StudioQuizSubmitDeps {
  /** Devuelve el quiz almacenado en el servidor, o null si no hay ninguno que calificar. */
  getStoredQuiz(userId: string, courseIdentifier: string, moduleIndex: number): Promise<StoredQuiz | null>;
}

export interface StudioQuizSubmitInput {
  userId: string;
  courseIdentifier: string;
  moduleIndex: number;
  /** El body crudo del request. Se parsea aquí a propósito: aquí estaba el hoyo. */
  body: unknown;
}

export type StudioQuizSubmitError = "respuestas_invalidas" | "sin_quiz_almacenado";

export type StudioQuizSubmitResult =
  | { ok: true; grade: QuizGrade }
  | { ok: false; error: StudioQuizSubmitError; message: string };

/** Extrae SOLO las respuestas del body. `score` no se lee jamás. */
export function parseAnswers(body: unknown): number[] | null {
  const answers = (body as { answers?: unknown } | null | undefined)?.answers;
  if (!Array.isArray(answers)) return null;
  if (!answers.every((a) => Number.isInteger(a))) return null;
  return answers as number[];
}

export async function submitStudioQuiz(
  deps: StudioQuizSubmitDeps,
  input: StudioQuizSubmitInput,
): Promise<StudioQuizSubmitResult> {
  const answers = parseAnswers(input.body);
  if (!answers) {
    return { ok: false, error: "respuestas_invalidas", message: "Respuestas requeridas" };
  }

  const stored = await deps.getStoredQuiz(input.userId, input.courseIdentifier, input.moduleIndex);
  if (!stored || stored.questions.length === 0) {
    console.error(
      `[studio-quiz] ${input.courseIdentifier}#${input.moduleIndex} (user=${input.userId}): ` +
      `sin quiz almacenado que calificar; intento RECHAZADO (nunca se auto-aprueba).`,
    );
    return {
      ok: false,
      error: "sin_quiz_almacenado",
      message: "No hay un quiz almacenado para este módulo, así que no podemos calificar tu intento. Genera el contenido del módulo e inténtalo de nuevo.",
    };
  }

  return { ok: true, grade: gradeQuizAttempt(stored.questions, answers, stored.passingScore) };
}
```

```ts
// Espera entre intentos REPROBADOS de quiz (spec 2026-07-17): 24 h. Corta el
// brute-force sin bloquear a quien sí sabe: intentos ilimitados, con espera.
// Un intento APROBADO nunca se bloquea — quien ya acreditó puede repetir el quiz
// cuando quiera. Se valida en el servidor: el cliente no es confiable.

export const QUIZ_COOLDOWN_MS = 24 * 60 * 60 * 1000;

export interface CooldownAttempt {
  passed: boolean;
  createdAt: Date | string;
}

export interface CooldownDecision {
  blocked: boolean;
  /** Milisegundos que faltan para poder reintentar. 0 si no está bloqueado. */
  retryAfterMs: number;
}

export function evaluateQuizCooldown(attempts: CooldownAttempt[], now: Date): CooldownDecision {
  const libre: CooldownDecision = { blocked: false, retryAfterMs: 0 };
  if (attempts.some((a) => a.passed)) return libre;

  const ultimoReprobado = attempts
    .filter((a) => !a.passed)
    .map((a) => new Date(a.createdAt).getTime())
    .reduce((max, t) => (t > max ? t : max), -Infinity);

  if (ultimoReprobado === -Infinity) return libre;

  const transcurrido = now.getTime() - ultimoReprobado;
  if (transcurrido >= QUIZ_COOLDOWN_MS) return libre;
  return { blocked: true, retryAfterMs: QUIZ_COOLDOWN_MS - transcurrido };
}
```

  /** El flag del curso para ESTE certificado: dc3_available o sep_available. */
  available: boolean;
  /** Intentos del socio (pueden venir de varios cursos: aquí se filtran). */
  attempts: CertAttemptSummary[];
}

export interface CertEligibilityDecision {
  eligible: boolean;
  state: CertEligibilityState;
}

export const CERT_LABEL: Record<PaidCertType, string> = {
  dc3: "constancia DC-3 STPS",
  sep: "constancia SEP",
};

const STATE_TEMPLATE: Record<CertEligibilityState, (label: string) => string> = {
  elegible: (l) => `Puedes solicitar tu ${l} de este curso.`,
  curso_no_encontrado: (l) => `Este curso no existe en el Tutor IA, así que no puede emitir ${l}.`,
  curso_sin_certificado: (l) => `Este curso no ofrece ${l}.`,
  sin_intento_aprobado: (l) => `Aprueba el quiz del curso en el Tutor IA para poder solicitar tu ${l}.`,
};

export function certMessage(state: CertEligibilityState, certType: PaidCertType): string {
  return STATE_TEMPLATE[state](CERT_LABEL[certType]);
}

export function evaluateCertEligibility(input: CertEligibilityInput): CertEligibilityDecision {
  if (!input.courseExists) return { eligible: false, state: "curso_no_encontrado" };
  if (!input.available) return { eligible: false, state: "curso_sin_certificado" };
  const aprobado = input.attempts.some(
    (a) => a.passed && a.courseIdentifier === input.courseIdentifier,
  );
  if (!aprobado) return { eligible: false, state: "sin_intento_aprobado" };
  return { eligible: true, state: "elegible" };
}

  );
